# Skill Analyzer

セッショントランスクリプト（jsonl）と SKILL.md を照合し、スキル実行時のエラー・仕様逸脱・改善点を自動的にリストアップするスキル。

## 実行手順

### ステップ 1: 対象スキルの特定

1. ユーザーがスキル名を指定している場合はそれを使用する
2. 指定がなければ、会話内で直近に実行されたスキル名を推定する
3. Glob ツールで `plugins/*/skills/*/SKILL.md` を検索し、対象スキルの SKILL.md パスを特定する

対象スキルが見つからない場合はユーザーに確認を求める。

### ステップ 2: セッショントランスクリプトの特定

1. ユーザーがセッション ID またはファイルパスを指定している場合はそれを使用する
2. 指定がなければ、直前のセッションを候補とする:

```bash
ls -t ~/.claude/projects/*/sessions/ | head -20
```

上記で最新の jsonl ファイルを取得する。候補が複数のプロジェクトにまたがる場合はユーザーに選択を求める。

### ステップ 3: Agent ツールで解析を実行

Agent ツール（general-purpose）に以下のプロンプトを渡して解析を実行する。トランスクリプトは大きくなりうるため、必ず Agent ツールに委譲して本体のコンテキストを保護すること。

```
以下の2ファイルを読み込み、スキル実行時のエラー・問題点を解析してください。

1. スキル定義: <SKILL.md パス>
2. セッションログ: <jsonl ファイルパス>

セッションログの読み方:
- jsonl 形式で1行1メッセージ
- スキル実行に関連するメッセージのみを解析対象とし、スキル実行前後の一般的な会話は無視する
- ツール呼び出しとその結果に注目する

解析観点:
- ツール呼び出しの失敗やエラー（Write 失敗、Edit 失敗など）
- SKILL.md で定義されたステップの実行順序の逸脱
- スキップされたステップ
- ループ条件・分岐条件の仕様違反（例: 条件を満たしているのに次のステップに進まない）
- タイムアウトやバックグラウンド移行などの非決定的動作
- エラーハンドリングフローが未検証の箇所

各イシューについて以下を報告:
1. 重大度（高/中/低）
2. 発生したステップ
3. 何が起きたか（トランスクリプトからの引用含む）
4. SKILL.md 上の期待動作
5. 改善案（SKILL.md の修正案）

最後にサマリテーブルを出力すること。日本語で報告。
```

### ステップ 4: 結果の報告

Agent ツールの結果をユーザーに報告する。

## 注意事項

- トランスクリプトは大きくなりうるため、必ず Agent ツールに委譲すること（本体のコンテキストで直接読まない）
